# i18n (cs + en, N-language ready) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Externalize all ~1,500–2,000 Czech UI strings into react-i18next catalogs (cs = reference, en = first translation), add per-user language, localize emails + invoice PDFs, and give the marketing site `/en/` routes — per the approved spec `docs/superpowers/specs/2026-07-12-i18n-and-payments-prep-design.md`.

**Architecture:** Frontend: i18next + react-i18next, JSON catalogs per feature namespace under `src/locales/{cs,en}/`, cs bundled eagerly (fallback), en lazy via dynamic import, TS key typing from the cs barrel. Backend: `app/core/i18n.py` catalog helper, per-locale Jinja2 email templates, invoice renderer parameterized by language. New `User.language` drives app UI + user-directed emails; `Organization.locale` keeps driving invoices/invitations.

**Tech Stack:** i18next, react-i18next, i18next-browser-languagedetector, i18next-resources-to-backend, Babel (already a backend dep), Alembic, Jinja2, WeasyPrint.

## Global Constraints

- Czech is the reference language and fallback (`fallbackLng: 'cs'`); vykání in all Czech copy. English: concise, professional SaaS tone (no "please kindly").
- `SUPPORTED_LANGUAGES = ['cs', 'en']` — defined ONLY in `frontend/src/lib/i18n/languages.ts` and `backend/app/core/i18n.py`.
- Catalog keys: camelCase, nested by page/component (`pipeline.addDeal`), plurals via i18next suffixes `_one/_few/_other` (cs) and `_one/_other` (en), interpolation `{{var}}`.
- After each batch: zero Czech literals left in the batch's files (verify with the diacritics grep below), tests green, catalog parity green.
- Brand words stay untranslated: SimpleCRM, Pipeline (nav label), QR Platba, ISDOC, ARES, IČO/DIČ (en labels explain them: "Company ID (IČO)").
- Backend commands on this Mac need: `DATABASE_URL=postgresql+asyncpg://simplecrm:simplecrm@localhost:5432/simplecrm POSTGRES_HOST=localhost DYLD_FALLBACK_LIBRARY_PATH=/opt/homebrew/lib` (CLAUDE.md).
- Frontend commands run from `frontend/`; backend from `backend/` with `uv run`.
- API types regen (after backend schema changes, server running): `BACKEND_OPENAPI_URL=http://localhost:8000/api/v1/openapi.json pnpm types:generate`.
- Playwright verification: both languages per UI batch; login/seed per `.claude/skills/running-simplecrm/SKILL.md`; screenshots to scratchpad only. ONE browser agent at a time.
- Diacritics grep (finds leftover Czech in code): `grep -rlP '[ěščřžýáíéúůťďňĚŠČŘŽÝÁÍÉÚŮŤĎŇ]' <paths> --include='*.ts' --include='*.tsx' | grep -v -E '(locales/|__tests__|\.test\.)'`

- Commit after every task (`git add` only the task's files).

## Externalization Procedure (referenced by Tasks 10–19, 21)

For each file in the task's scope:
1. Find every user-facing literal: JSX text nodes, `aria-label`/`title`/`placeholder` attrs, toast calls, `setError`/validation strings, `usePageTitle("…")` args, enum-label map values, chart labels, empty states. Developer-only strings (console, test ids, keys) stay.
2. Add keys to `frontend/src/locales/cs/<ns>.json` (value = existing Czech text verbatim, vykání preserved) and `frontend/src/locales/en/<ns>.json` (your translation). Group keys by component: `{"dealsList": {"title": "Obchody", …}}`.
3. Replace the literal: `const { t } = useTranslation("<ns>");` → `t("dealsList.title")`. For counts, ONE key with plural suffixes replaces `csPlural`/`csNoun`/ternaries: cs `"dealCount_one": "{{count}} obchod", "dealCount_few": "{{count}} obchody", "dealCount_other": "{{count}} obchodů"`; en `"dealCount_one": "{{count}} deal", "dealCount_other": "{{count}} deals"`; call `t("dealCount", { count })`.
4. Enum-label maps become key maps typed against the catalog: `const ROLE_KEY = { admin: "roles.admin", manager: "roles.manager", salesperson: "roles.salesperson" } as const satisfies Record<UserRole, ParseKeys<"settings">>;` (import `ParseKeys` from i18next) → render `t(ROLE_KEY[role])`.
5. Non-component modules (e.g. `settingsNav.ts` labels) store keys, not text; the rendering component translates.
6. Run: `pnpm typecheck && pnpm test -- run <batch test files> && node scripts/check-catalog-parity.mjs` then the diacritics grep on the batch paths — must return nothing.
7. Playwright: navigate the batch's routes logged in as the seeded admin, screenshot (cs), switch language to en via the app switcher, screenshot, check console for errors and untranslated-key leaks (`missingKey` warnings).
8. Commit: `git commit -m "i18n(<area>): externalize copy to cs/en catalogs"`.

- [ ] **Step 7:** Create `scripts/check-catalog-parity.mjs`:

```js
import { readFileSync, readdirSync } from "node:fs";
import path from "node:path";

const ROOT = path.resolve(import.meta.dirname, "../src/locales");
const REF = "cs";
const PLURAL_SUFFIX = /_(zero|one|two|few|many|other)$/;

function flatten(obj, prefix = "", out = new Set()) {
  for (const [k, v] of Object.entries(obj)) {
    const key = prefix ? `${prefix}.${k}` : k;
    if (v && typeof v === "object") flatten(v, key, out);
    else out.add(key.replace(PLURAL_SUFFIX, ""));
  }
  return out;
}

const locales = readdirSync(ROOT, { withFileTypes: true })
  .filter((d) => d.isDirectory())
  .map((d) => d.name);
const refFiles = readdirSync(path.join(ROOT, REF)).filter((f) => f.endsWith(".json"));
const errors = [];

for (const locale of locales.filter((l) => l !== REF)) {
  for (const file of refFiles) {
    const refKeys = flatten(JSON.parse(readFileSync(path.join(ROOT, REF, file), "utf8")));
    let target;
    try {
      target = JSON.parse(readFileSync(path.join(ROOT, locale, file), "utf8"));
    } catch {
      errors.push(`${locale}/${file}: missing file`);
      continue;
    }
    const targetKeys = flatten(target);
    for (const k of refKeys) if (!targetKeys.has(k)) errors.push(`${locale}/${file}: missing "${k}"`);
    for (const k of targetKeys) if (!refKeys.has(k)) errors.push(`${locale}/${file}: orphan "${k}" (not in ${REF})`);
  }
}

if (errors.length) {
  console.error(`Catalog parity FAILED (${errors.length}):\n` + errors.join("\n"));
  process.exit(1);
}
console.log(`Catalog parity OK (${locales.length - 1} locale(s) vs ${REF}).`);
```

Add to `package.json` scripts: `"i18n:check": "node scripts/check-catalog-parity.mjs"`.

def language_for_locale(locale: str | None) -> str:
    """'cs-CZ' -> 'cs'; unknown/missing -> DEFAULT_LANGUAGE."""
    if not locale:
        return DEFAULT_LANGUAGE
    base = locale.split("-")[0].split("_")[0].lower()
    return base if base in SUPPORTED_LANGUAGES else DEFAULT_LANGUAGE

async def patch_my_language(
    payload: LanguageUpdate,
    user: User = Depends(get_current_user),
    session: AsyncSession = Depends(get_db),
